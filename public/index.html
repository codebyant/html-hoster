Como funciona

Publique HTML gerado por IA sem deploy manual

Envie HTML, defina o path e receba uma URL pública pronta para compartilhar imediatamente.

01

Defina o Path

Escolha o slug ou path público desejado (ex: /my-page). Sem subdomínios ou configurações complexas.

02

Envie o HTML

Publique via API, CLI ou use seu agente de IA (via MCP) integrado com sua API key configurada por env.

03

Disponível Imediatamente

O servidor valida, salva no storage (local ou S3) e serve o HTML estático instantaneamente. Sem painel ou builds.

Agente de IA
API / cURL
$ curl -X POST https://api.seusite.com/pages \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "path": "/my-page",
    "html": "<main><h1>my-page</h1></main>"
  }'

⬇ Response:
{
  "success": true,
  "url": "https://html.meusite.com/my-page"
}

Funcionalidades

Tudo que você precisa, nada além

Projetado para ser a camada de publicação de qualquer pipeline que gere HTML — manualmente ou por IA.

🔑

Autenticação por API Key

Todas as rotas de escrita exigem Authorization: Bearer. Chave configurada via variável de ambiente.

☁️

Storage local ou S3

Filesystem em desenvolvimento, S3 compatível (AWS, MinIO, R2) em produção. Camada abstrata intercambiável via STORAGE_DRIVER.

🤖

MCP nativo para Claude

Servidor MCP embutido. Claude Desktop e Claude Code enxergam publish_page, delete_page e mais como tools nativas.

📤

Upload de arquivo

Além do JSON, aceita upload de .html via multipart/form-data. Útil para pipelines com arquivos físicos.

🛡️

Path seguro

Validação rígida contra path traversal, segmentos inválidos e prefixos reservados. Nenhum HTML vindo da API executa lógica server-side.

🌗

Modos dev, demo e production

Swagger visível só em dev/demo. Modo demo desativa writes e storage — ideal para mostrar o projeto sem risco.

📄

Swagger integrado

Documentação OpenAPI 3.1 gerada automaticamente em /api/docs. Spec disponível em JSON para integração com qualquer ferramenta.

⚡

Bun + Hono

Runtime rápido, TypeScript nativo, zero build step necessário em dev. Deploy em qualquer lugar que rode Bun.

API REST

Endpoints

Interface HTTP simples. Qualquer linguagem, ferramenta externa ou IA com acesso HTTP consegue usar.

GET /api/pages Lista todas as páginas publicadas 🔒 auth
PUT /api/pages/:path Publica ou atualiza uma página via JSON 🔒 auth
POST /api/pages/:path Publica via upload de arquivo (multipart) 🔒 auth
GET /api/pages/:path Retorna metadados da página 🔒 auth
DELETE /api/pages/:path Remove a página publicada 🔒 auth
GET /:path Serve o HTML publicamente público
Publicar
Upload arquivo
Deletar
# Publicar ou atualizar
curl -X PUT https://html.seusite.com/api/pages/my-page \
  -H "Authorization: Bearer sua-chave" \
  -H "Content-Type: application/json" \
  -d '{"html":"<!DOCTYPE html><html><body><h1>my-page</h1></body></html>"}'

{ "success": true, "url": "https://html.seusite.com/my-page", "path": "my-page" }
# Upload de arquivo
curl -X POST https://html.seusite.com/api/pages/my-page \
  -H "Authorization: Bearer sua-chave" \
  -F "file=@minha-pagina.html"

{ "success": true, "url": "https://html.seusite.com/my-page", "path": "my-page" }
# Deletar
curl -X DELETE https://html.seusite.com/api/pages/my-page \
  -H "Authorization: Bearer sua-chave"

{ "success": true, "path": "my-page" }

INTEGRAÇÃO COM IA

MCP nativo para qualquer agente compatível

Configure o servidor MCP uma vez e permita que sua IA publique, atualize e gerencie páginas HTML usando ferramentas estruturadas.

Ferramentas MCP prontas para uso

Seu agente não precisa improvisar chamadas HTTP. Ele recebe tools tipadas para publicar, listar, consultar e remover páginas com segurança.

Ver configuração →

Compatível com: Claude Desktop, Claude Code, Cursor, clientes MCP customizados e outros agentes compatíveis.

publish_page Publica ou atualiza HTML em um path.
get_page_meta Retorna URL, metadata e data de atualização.
delete_page Remove uma página publicada.
list_pages Lista páginas publicadas e suas URLs.
Configuração em um cliente MCP

O exemplo abaixo demonstra a integração com o Claude Desktop. O servidor funciona de forma idêntica em qualquer cliente compatível. 

{
  "mcpServers": {
    "html-hoster": {
      "command": "npx",
      "args": [
        "-y",
        "mcp-remote@latest",
        "https://html.seusite.com/mcp",
        "--header",
        "Authorization:${HTML_HOSTER_AUTH}"
      ],
      "env": {
        "HTML_HOSTER_AUTH": "Bearer sua-chave"
      }
    }
  }
}
